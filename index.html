Miniature PHP

PHP Micro Framework

Rahil Khurshid Ali
A Micro Framework For Everyday PHP Programming
Miniature is a very small PHP Framework, which purpose is to provide developer the ease of creating web applications fast, by writing less and efficient code.
Inspired from CodeIgniter, Miniature tends to provide a set of useful classes that can be used to write fast and easy web application.

Key Features
  • An Object-Oriented Framework.
  • CodeIgniter like Database Abstraction Layer and Form Validation System.
  • Loose coupled.
  • Lightweight and fast.
  • Easy configuration.
  • Modifiable Classes.

Setting Up the Project

Preparing Files

In every php file that you create inside the root directory of the project, it must include the init.php file which is located inside the includes folder.
To do so, require once the file. 
require_once('includes/init.php');

Setting Up the Database Connection

In the "includes" folder, open "init.php"
The Database credentials can be set in the $GLOBALS['config'] array. 
$GLOBALS['config'] = array(
    'mysql' => array(
        'host' => 'hostname',
        'username' => 'your_username',
        'password' => 'your_password',
        'db_name' => 'database_name'
    )
)
Enter the database connection values into their respective keys, like
'host' => 'localhost' // and so on...

Configuration Basics

The Miniature framework configuration is easy and modifiable.
The system relies on the $GLOBALS['config'] array in the init.php file. All the configuration options can be set there. 
$GLOBALS['config'] = array(
    'mysql' => array(
        'host' => 'your_host_name',
        'username' => 'your_username',
        'password' => 'your_password',
        'db_name' => 'your_db_name'
        ),
    'remember' => array(
        'cookie_name' => 'hash',
        'cookie_expiry' => 604800
        ),
    'session' => array(
        'user_session' => 'user',
        'token_name' => 'token'
        ),
    'redirect' => array(
        '404' => 'includes/404.php'
        ),
    'table_name' => array(
        'users' => 'users',
        'user_session' => 'user_session'
        ),
    'table_col' => array(
        'user_session' => array(
            'hash' => 'hash'
            )
        )
);
The system heavily uses this array structure to access different credentials.
Like, if database connection has to access the hostname, it can do so by navigating to 'mysql' array to find the 'host' key.
If the User Class has to access the user's table name, it does that by navigating to 'table_name' and inside it the 'users' key which stores the user table name.
The array can go deeper to granulize the site options.

For example you want a place to store information about multiple directories inside the project, you can do this by creating a custom array for directories inside the $GLOBALS['config'] array.
'directory' => array(
    'uploads_dir' => 'uploads',
    'images_dir' => 'uploads/images'
)
Then all you have to do is to access the directory array to retrieve the respective path. And this is discussed in the Accessing Configuration Options section

Accessing Configuration Options

Accessing configuration values is easy.
Taking the previous directory example;
'directory' => array(
    'uploads_dir' => 'uploads',
    'images_dir' => 'uploads/images'
)
Accessing: And which can be accessed like this, 
Config::get('directory/uploads_dir'); // outputs 'uploads'
Similarly, for accessing the database name you can write,
Config::get('mysql/db_name'); // outputs 'database_name'
The array can go much deeper. Take the 'table_col' array as an example which stores column names for a particular table.
'table_col' => array(
    'user_session' => array(
        'hash' => 'hash',
        'user_id' => 'uid'
        )
)
The hash column name can be accessed by, 
Config::get('table_col/user_session/hash'); // outputs 'hash'

Passing Paramters

The query method provides the ease of binding paramters with query, instead of contactinating them, which causes serious security issues.
Syntax: 
$db->query(‚ÄúSELECT * FROM table_name WHERE column_name = ?‚Äù, array(value))->getResult();
The "?" sign in the query is the placeholder for the value, i.e. when the query is executed, the placeholder is replaced with the value.
The following example demonstrates how to bind values into a query.
Demo: 
$db->query(‚ÄúSELECT * FROM blog WHERE `id` > ? AND `id` < ?‚Äù, array(2, 6))->getResult();
The two placeholder will be replaced with numeric values in the array, respect to their position, so the index position of the values matter.
Here is an another example of retrieving a record by matching a string value.
Demo: 
$db->query("SELECT * FROM blog WHERE title IN (?)", array('Birds of Fire'))->getResult();

Instantiating Input Class

With the Input Class, you can,
  • get sanitized POST/GET values.
  • sanitize inputs.
  • sanitize an array of input values.
Input Class does not need to be instantiated as it uses Static methods.

Form Input


Reading POST values from a Form:
Instead of reading a POST input value like this; 
$_POST['input_name'];
Input Class provides a method, which reads and sanitizes the input at the same time.
Syntax: 
Input::get('input_name');
returns sanitize form input value, otherwise blank value, if no input name was provided.
Demo: Suppose there is a form input, 
<form action="form.php" method="POST">
    <input name="username" type="text"/>
</form>
The POST value can be retrieved like this; 
$username = Input::get('username'); // returns the username

Reading GET values from a query string:
Demo: an example query string contcatenated from a URL. 
http://www.examplesite.com/contact_form.php?email=youremail@domain.com&message=this%20is%20a%20message
The GET value can be retrieved like this; 
$email = Input::get('email');  // returns youremail@domain.com
$message = Input::get('message'); // returns this is a message

Checking Input Values:

The exists(); method checks if a certain POST/GET value already exists.
Syntax: 
Input::exists('input_name');
returns true if exists, otherwise false.
Demo:
if(Input::exists('username')){
    echo Input::get('username');
}

Sanitizing Single Input

The Input::sanitize('input_name'); sanitizes any input.
Demo: 
Input::sanitize('username'); // returns sanitized username

Sanitizing The Entire POST/GET Array

The $_POST array returned upon a form submission or $_GET array from a query string can entirely be sanitized using the sanitizeAttributes(); method.
Syntax: 
Input::sanitizeAttributes('input_type');
if no parameter is passed, $_POST array is sanitized by default, otherwise pass 'get' to sanitize $_GET array 
Input::sanitizeAttributes('get'); //returns sanitized GET array of values.

Instantiating Validation Class

Validation Class provides a comprehensive system which can be use to validate form input without writing more than 100 lines of code.
To use the Validation Class, it needs to be instantiated; 
$validation = new Validation();

Validating Form Input

Validation is easy as never before!
Only a proper demonstration will convince you.
Demo: Here is a simple registration form; 
<form action="register.php" method="POST">
      Username: <input name="username" type="text"/>
      Password: <input name="password" type="password"/>
      Confirm Password: <input name="confirm_password" type="password"/>
      Email: <input name="email" type="text"/>
      Full Name<input name="full_name" type="text"/>
      <input type="submit" value="Register" />
</form>
to validate the form input, upon submission; 
$validation = new Validation(); //create a new Validation object
$validation->check($_POST, array(
    'username' => array(
        'required' => true,
        'min' => 6,
        'max' => 30,
        'unique' => 'users'
        ),
    'password' => array(
        'required' => true,
        'min' => 8,
        'max' => 60
        ),
    'confirm_password' => array(
        'required' => true,
        'matches' => 'password'
        ),
    'email' => array(
        'required' => true,
        'valid_email' => true
        ),
    'full_name' => array(
        'required' => true,
        'min' => 2,
        'max' => 128
        )
    ));
Syntax: 
$validation = new Validation();
$validation->check(source, array(
    'input_name' => array(
        'rule_name' => 'rule_value'
    )
));
In the above demonstration, $_POST was the source, so it was passed as the first parameter. To handle the errors generated from valiation, check "Handling Validation Errors" section.

Validation Rules

How to apply validation rules Pass an array of inputs with their respective rules as the second parameter of the check() method. 
array(
    'field_name' => array ( 
        //set of rules comes here

        'rule1' => 'rule_value1',
        'rule2' => 'rule_value2'

        // and so on....
    )
)

Rules:

required: array( 'required' => true ) will strict a form input to be required.
max: array( 'max' => numeric_value ) checks if a certain form input has a maximum length greater than numeric_value.
min: array( 'min' => numeric_value ) checks if a certain form input has a length minimum than numeric_value.
unique: array( 'unique' => table_name ) checks if a certain form input value already exists in the table.
matches: array( 'matches' => input_name ) checks if a certain inputs matches another field in the form as provided in the rules array.
numeric: array( 'numeric' => true ) checks if a certain form input is a numeric value.
valid_email: array( 'valid_email' => true ) checks if an email input is a valid email.

Session Class

Instantiating Session Class

Using Session Class, you can:
  • Get/Set Session variable.
  • Delete/Unset Session variable.
  • Flash a Session.
  • Check if a Session exists.
As the Session Class uses static class methods, so it not needs to be instantiated.

Setting a Session

Syntax: 
Session::set($session_name, $session_value);
returns $_SESSION[$session_name] = $session_value
Demo: 
Session::set('user_data', 'john')
Result: 
$_SESSION['user_data'] = 'john'

Getting a Session

Syntax: 
Session::get($session_name);
returns the value of the session name.
Demo: 
echo Session::get('user_data');
Result: 
john

Deleting a Session

Syntax: 
Session::delete($session_name);
returns true if the session existed and successfully deleted, otherwise false if session of no such name exists.
Demo: 
Session::delete('user_data');

Flashing

Flashing a session, creates a session and then deletes it on the second method call.
The method is useful for displaying alert messages.
Syntax: 
Session::flash($session_name, $session_value); //to set the session
Session::flash($session_name); //to call and unset the session on the fly.
returns the session value if it exists and flashes it value, otherwise set a new session.
Demo: 
Session::flash('success_messsage', 'The post has been deleted'); //sets a session
echo Session::flash('success_messsage'); // echos 'The post has been deleted'
echo Session::flash('success_messsage'); // echos nothing on the second call, because the session has been flashed.

Check if a Session exists

The Session::exists($session_name) checks if the session exists.
Syntax: 
Session::exists($session_name)
returns true if exists otherwise false.
Demo: 
if(Session::exists('user_data')){
    echo Session::flash('user_data');
}

Token Class

Token Class

  • Instantiating Class
  • Integration into Forms

Instantiating Token Class

The Token Class provides security for avoiding Cross Site Request Forgery (CSRF) where unauthorized commands are transmitted from a user that the website trusts.
For example, a user try to submit a form by passing malicious values through URL query string, instead of hitting the submission button.
This is where the Token Class comes in for the rescue.
As the Token Class uses static class methods, so it not needs to be instantiated.

Integration into Forms

Here the example of an attack on a web form is taken to demonstrate the security provided by the Token Class.
<form action="register.php" method="POST">
      Username: <input name="username" type="text"/>
      Password: <input name="password" type="password"/>
      Confirm Password: <input name="confirm_password" type="password"/>
      Email: <input name="email" type="text"/>
      Full Name<input name="full_name" type="text"/>
      <input type="submit" value="Register" />
</form>
Certain user tries to hit the form by passing value through query string. 
http://www.somedomain.com/register.php?username=45ffsdfe&password=567345ty&confirm_password=567345ty&email=fdrevv@454.com&full_name=ewrf%20sdfsc%20sdfsdf
Therefore, the Token Class provides two methods to avoid such attack.
Token::generate();
Token::check('token_name');

You need to call and echo out the Token::generate() method in the value of an input field of type hidden. It is shown in bold in the form.
<form action="register.php" method="POST">
      Username: <input name="username" type="text"/>
      Password: <input name="password" type="password"/>
      Confirm Password: <input name="confirm_password" type="password"/>
      Email: <input name="email" type="text"/>
      Full Name<input name="full_name" type="text"/>
      <input name="token" type="hidden" value="<?php echo Token::generate(); ?>"/>   <-- the Token::generate() method is integrated into a form as an hidden input -->
      <input type="submit" value="Register" />
</form>
Then upon form submission, the token generated in the form is needed to be checked. 
if(Token::check(Input::get('token'))){ // where 'token' is the form input name
    // all of the form validation and processing code comes inside this check.
}
The proper usage of Token Class is demonstrated in the Login/Registration Form application demo.

Login/Logout System

The use of User Class is actually to create a secure an easy login n registration system, which is demonstrated in this section.
The system utilizes, the Token, Input, Validate and User classes.

Logging in a user includes the following steps:
  1. Checking for the form if it was submitted or not?
  2. Checking if the $_POST array is not empty.
  3. Checking for valid token, if CSRF protection is applied in the form.
  4. Validating the input from the form.
  5. If the validation was successful, Instantiate the User Class.
  6. Login the User, using the User Class login() method.
Login Form 
<form action="login.php" method="POST">
      Username: <input name="username" type="text"/>
      Password: <input name="password" type="password"/>
      <input name="remember" type="checkbox" value="on" /> Remember Me
      <input name="token" type="hidden" value="<?php echo Token::generate(); ?>" /> <-- remember to include the token input -->
      <input type="submit" value="Log In" />
</form>
Form Processing 
<?php
// include the init.php file
require_once 'includes/init.php';

// check if $_POST is not an empty array
if(Input::exists()){

    // check for a valid token
    if(Token::check(Input::get('token'))){

        // validating inputs
        $validate = new Validation();
        $validate->check($_POST, array(
            'username' => array( 'required' => true  ),
            'password' => array( 'required' => true  )
        ));

        // check if the validation was successful
        if($validate->getPassed()){

            // create a new user
            $user = new User();

            // getting the remember me value, if it was checked
            $remember = (Input::get('remember') === 'on') ? true : false;

            // calling the login method and passing username and password to authenticate the user.
            // if remember me checkbox was checked, then pass it input value in the login method as the 3rd parameter.
            if($user->login(Input::get('username'), Input::get('password'), $remember)){

                // redirect to main page on authorizing a valid user.
                Redirect::to('index.php');
            }else{

                //otherwise output login error
                // the Validation Class addError() method can be used to add more errors to the Errors array.
                $validate->addError("Invalid Username/Password");
                $errors = $validate->getErrors();
            }
        } else{

            // the $errors[] array then can be display as HTML
            $errors = $validate->getErrors();
        }
    }
}
?>

        $validate = new Validation();

            'username' => array(
                'required' => true,
                'min' => 4,

                'required' => true,

            'full_name' => array(
                'required' => true,
                'min' => 2,
                'max' => 128
            )
        ));

        // check if the validation was successful
        if($validate->isPassed()){

            // create a new user
            $user = new User();

            // generating a salt
            $salt = Hash::generateSalt(32);

            // insert the user into the table
            try{
                $userID = $user->save(array(

            }

}

Storing User's Cookie

The 'Login/Logout System' section, integrates the 'Remember Me' system into the login form, which stores a user's cookie, and logs it in the next time the users open the site.
The section gets into the detail of how to Store User's Cookie.

You need to create a 'User Session' table which can store the user cookies.
  1. Create a User Session table, e.g tbl_user_session.
  2. The table structure must look like this
    Column Name Type
    id INT (PK)
    uid INT (FK)
    hash VARCHAR(255)
  3. Open the init.php file and inside the 'table_name' array, add a key value pair of user session table where the key is the reference to the user session table and the value is the name of that table. Be Careful about it! 
    'table_name' => array(
    'user_session' => 'tbl_user_session'
)
// reference to the table => 'table name in the database'
  4. And inside the 'table_col' array add the user session column names. The keys must be exactly named like in the code below. 
    'table_col' => array(
    'user_session' => array(
        'hash' => 'hash',
        'user_id' => 'uid'
    )
)
// 'reference to column' => 'column name in the table'
    We are now done with the configuration here.
  5. In the login form, provide a checkbox input for remember me and set it's value to 'on'. 
    <input name="remember" type="checkbox" value="on" /> Remember Me
  6. In the form processing, store the state of remember me input. Storing TRUE will activate cookie. 
    $remember = (Input::get('remember') === 'on') ? true : false;
  7. Finally, upon loggin in the user using the login() method, pass the $remember bool variable as the 3rd parameter. 
    $user->login(Input::get('username'), Input::get('password'), $remember);
This is all you need to setup a system for Storing User's Cookie
